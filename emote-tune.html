Overview

EmoteTune is an intelligent music recommendation platform that analyzes a user's facial expressions in real time, detects emotion using deep learning, and recommends personalized music based on emotion, language, and listening intensity. It turns a webcam feed into a feeling-aware DJ — closing the gap between how someone feels and what they hear next.

Mood-aware recommendation matters because static playlists don't adapt to the listener. By reading emotion directly from expression, EmoteTune reduces the friction of manually picking a song and opens up real-world use cases in wellness apps, smart speakers, and in-car entertainment systems.

What it does

  • Detects 7 human emotions in real time from webcam frames using DeepFace and OpenCV.
  • Recommends songs by filtering a curated dataset on emotion, language, and intensity, with graceful fallbacks when an exact match isn't available.
  • Accepts voice commands for hands-free playback control via the Web Speech API.
  • Visualizes emotion history over time on an interactive analytics dashboard.
  • Offers an AI mood chatbot for lightweight conversational wellness tips.
  • Supports four languages — English, Hindi, Tamil, and Malayalam.

Challenges & Learnings

Building a real-time, camera-driven pipeline meant fighting latency, lighting, and accuracy problems at once. Hover the cards to reveal how each was solved:

⚡

Compute Cost

Running emotion detection on every frame was too slow for real-time use.

01 // LATENCY

Core Solution

Added frame throttling so inference runs on every 20th frame, keeping the UI responsive without losing detection quality.

01 // SOLVED
💡

Poor Lighting

Low light or backlit webcam feeds caused emotion prediction to fail outright.

02 // ROBUSTNESS

Core Solution

Added stronger face-detection preprocessing plus a fallback path so the app degrades gracefully instead of crashing.

02 // SOLVED
🎯

Match Accuracy

Emotion-only filtering often returned recommendations that felt off.

03 // ACCURACY

Core Solution

Designed emotion + language + intensity filtering with layered fallback rules so a relevant song is always returned.

03 // SOLVED
🌐

Multilingual Voice

Voice commands needed to work across four different languages reliably.

04 // I/O

Core Solution

Layered the Web Speech API with language detection and retry logic to keep recognition stable across accents.

04 // SOLVED
