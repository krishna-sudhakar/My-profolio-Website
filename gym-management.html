Overview

The Gym Management System digitizes day-to-day fitness center operations that are traditionally handled on paper — member registration, trainer task assignment, and equipment upkeep. It replaces scattered manual records with one centralized platform built around three roles: Admin, Trainer, and User, each with its own dashboard and permissions.

The system is built on PHP and MySQL for the backend and data layer, with HTML, CSS, and JavaScript powering a responsive frontend — a deliberately lightweight, dependency-free stack chosen to keep the project affordable and easy to maintain for small to medium-sized gyms.

What it does

  • Role-based login and dashboards for Admin, Trainer, and User, each scoped to its own permissions.
  • Admins add or remove trainers and users, and manage the equipment inventory.
  • Trainers search for a member, assign weekly workout tasks, and track completion status.
  • Users view their profile and see the tasks assigned to them, marked complete as they progress.
  • Registration captures height, weight, and computes BMI in real time on the client side.
  • Equipment records track working/maintenance status and last maintenance date.

Challenges & Learnings

Most of the hard problems here were about access control and data integrity rather than algorithms. Hover the cards to see how each was handled:

🔐

Role-Based Access

Three distinct roles needed strictly separated permissions and views.

01 // RBAC

Core Solution

Implemented role-based access control so admins, trainers, and users only ever see dashboards and data scoped to their own role.

01 // SOLVED
🛡️

Data Security

A login system handling credentials is a direct target for SQL injection and XSS.

02 // SECURITY

Core Solution

Used prepared statements for all queries, input validation, and password hashing to close off common injection and scripting attacks.

02 // SOLVED
⏱️

Session Handling

Idle sessions left logged in are a security risk across shared gym terminals.

03 // SESSIONS

Core Solution

Added session management with auto-logout on inactivity and activity logging to keep every session accountable.

03 // SOLVED
🗃️

Referential Integrity

Users, trainers, and equipment all reference each other, risking orphaned records.

04 // DATA

Core Solution

Normalized the schema and enforced relationships with foreign keys across the admin, user, trainer, and equipment tables.

04 // SOLVED
